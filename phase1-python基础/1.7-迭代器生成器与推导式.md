## 核心问题：什么时候该"懒"？

C# 有 LINQ 的延迟执行（`IEnumerable<T>` 的惰性求值，直到被 `ToList()` 或 `foreach` 消费）。
Python 也有完全相同的概念——迭代器和生成器。

**核心思想：不要一次性生成所有数据，按需计算。**

## 1. for 循环的底层机制

### `for x in obj` 实际上调用了什么？

```python
# 你写的：
for item in [1, 2, 3]:
    print(item)

# Python 实际上做了：
iterator = iter([1, 2, 3])    # 1. 调用 __iter__() 获取迭代器
while True:
    try:
        item = next(iterator)  # 2. 不断调用 __next__() 获取下一个
        print(item)
    except StopIteration:       # 3. 捕获 StopIteration 终止循环
        break
```

**这就是迭代器协议：**

```
对象有 __iter__() → 返回迭代器
迭代器有 __next__() → 返回下一个元素，或抛 StopIteration
```

### 实现可迭代对象

```python
class CountDown:
    def __init__(self, start):
        self.start = start

    def __iter__(self):
        # 每次 for 循环时调用，返回一个迭代器
        return CountDownIterator(self.start)

class CountDownIterator:
    def __init__(self, start):
        self.current = start

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

# 使用
for i in CountDown(5):
    print(i)  # 5, 4, 3, 2, 1
```

## 2. `__iter__` 和 `__next__`

### 更简单的写法：一个类实现两个方法

```python
class CountDown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        self.current = self._start  # 重置
        return self                 # 自己就是迭代器

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value
```

**对比 C#：** C# 的 `IEnumerable<T>` + `IEnumerator<T>` 也是两个接口，和 Python 的 `__iter__` + `__next__` 一一对应。

```csharp
public class CountDown : IEnumerable<int>
{
    public IEnumerator<int> GetEnumerator() => new CountDownEnumerator(Start);
    // Python 的 __iter__ 对应 GetEnumerator()
}
```

## 3. `yield` 和生成器

### 为什么生成器比列表高效？

```python
# 列表版本：一次性生成所有值
def generate_numbers_list(n):
    result = []
    for i in range(n):
        result.append(i * 2)
    return result

# 生成器版本：按需生成
def generate_numbers_gen(n):
    for i in range(n):
        yield i * 2

# 测试
nums_list = generate_numbers_list(1_000_000)  # 立即占用 ~8MB
nums_gen = generate_numbers_gen(1_000_000)     # 几乎不占内存

# 访问第一个元素
nums_list[0]  # 直接读取
# nums_gen[0]  # ❌ 生成器不支持索引！
next(nums_gen)  # ✅ 生成器只能逐个取
```

**核心区别：**

| 版本 | 内存 | 速度 | 能否索引 |
|------|------|------|---------|
| 列表（eager） | 全部生成在内存 | 快（缓存友好） | ✅ |
| 生成器（lazy） | 一次一个 | 略慢（每次 yield 有开销）| ❌ |

**惰性求值的意义：**

```python
# 处理大文件——生成器版本
def read_large_file(filename):
    with open(filename) as f:
        for line in f:
            yield line.strip()

for line in read_large_file("huge_data.txt"):
    process(line)  # 一次只处理一行，不把整个文件加载到内存
```

### yield 的工作原理

```python
def simple_gen():
    print("第一步")
    yield 1
    print("第二步")
    yield 2
    print("第三步")

gen = simple_gen()        # 还没有执行任何代码
print(next(gen))          # "第一步" → 1
print(next(gen))          # "第二步" → 2
print(next(gen))          # "第三步" → StopIteration
```

`yield` 像是一个「暂停点」——每次 `next()` 从上次暂停的地方继续执行，直到下一个 `yield`。

## 4. 推导式（Comprehension）

### 列表推导式

```python
# 传统写法
squares = []
for i in range(10):
    squares.append(i ** 2)

# 推导式
squares = [i ** 2 for i in range(10)]
```

### 带条件的推导式

```python
# 偶数平方
evens_squared = [i ** 2 for i in range(20) if i % 2 == 0]
# [0, 4, 16, 36, 64, ...]

# 多重循环
pairs = [(x, y) for x in range(3) for y in range(3)]
# [(0,0), (0,1), (0,2), (1,0), (1,1), (1,2), (2,0), (2,1), (2,2)]
```

### 字典推导式和集合推导式

```python
# 字典推导式
squares_dict = {i: i ** 2 for i in range(5)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 4}

# 集合推导式
unique_lengths = {len(word) for word in ["hello", "world", "python", "code"]}
# {3, 5, 6}
```

### 对比 C# 的 LINQ

```csharp
// C# LINQ
var squares = Enumerable.Range(0, 10)
    .Select(i => i * i)
    .ToList();

// Python 推导式
squares = [i * i for i in range(10)]
```

LINQ 是**方法链**风格，推导式是**类数学集合**风格。LINQ 更灵活（更多操作符），推导式更简洁。

## 5. 生成器表达式

### `(x for x in range(10))` 节省内存

```python
# 列表推导式：生成全部元素
list_comp = [i * 2 for i in range(1_000_000)]  # 立即占内存

# 生成器表达式：惰性求值
gen_expr = (i * 2 for i in range(1_000_000))   # 几乎不占内存

# 使用
for val in gen_expr:
    if val > 100:
        break
    print(val)
# 只生成了前 51 个值，其余的根本没有生成
```

**语法区别：** 方括号 `[]` = 列表推导式，圆括号 `()` = 生成器表达式。

**使用场景：** 当数据量大或只需部分结果时，生成器表达式节省内存。

```python
# 求前1000个数的平方和
sum(i ** 2 for i in range(1000))  # 生成器表达式，不用创建列表
```

## 6. itertools 常用工具

```python
import itertools

# chain：串联多个可迭代对象
list(itertools.chain([1,2,3], [4,5,6]))  # [1,2,3,4,5,6]

# count：无限计数
for i in itertools.count(10):  # 10, 11, 12, ...
    if i > 15: break
    print(i)

# cycle：无限循环
count = 0
for item in itertools.cycle(["A", "B", "C"]):
    print(item)
    count += 1
    if count > 6: break  # A, B, C, A, B, C, A

# groupby：分组（和 SQL 的 GROUP BY 类似，但要求数据已按分组键排序）
data = [("A", 1), ("A", 2), ("B", 3), ("B", 4)]
for key, group in itertools.groupby(data, key=lambda x: x[0]):
    print(key, list(group))  # A [(A,1),(A,2)]  B [(B,3),(B,4)]
```
