## 核心问题：函数为什么能这么灵活？

C# 的重载、可选参数、`params` 关键字是**编译时决定**的。
Python 却是**运行时决定**——函数本身就是对象。

这个差异解释了 Python 函数的几乎所有"奇怪"行为。

## 1. `*args` 和 `**kwargs`：可变参数

### 为什么需要这两种参数？

C# 有 `params` 关键字让你传任意数量的同类型参数：

```csharp
int Sum(params int[] numbers) { ... }
Sum(1, 2, 3);
```

Python 的 `*args` 做同样的事，但更强大——还支持**关键字参数集合**。

```python
# *args：收集剩余位置参数为 tuple
def log(message, *tags):
    print(f"[{', '.join(tags)}] {message}")

log("系统启动", "INFO", "网络")   # [INFO, 网络] 系统启动
log("启动成功")                     # [] 启动成功（tags 是空 tuple）

# **kwargs：收集剩余关键字参数为 dict
def create_user(name, **info):
    print(f"用户: {name}")
    for key, value in info.items():
        print(f"  {key}: {value}")

create_user("Alice", age=30, role="admin", email="a@b.com")
```

### "*"到底做了什么？

```python
# 定义时的 *：收集
def f(*args):
    print(args)

f(1, 2, 3)              # (1, 2, 3) —— 三个位置参数被收集

# 调用时的 *：展开
def f(a, b, c):
    print(a, b, c)

nums = [1, 2, 3]
f(*nums)                 # 1 2 3 —— 列表被展开成三个参数

# 同样的逻辑适用于 **（展开字典）
def greet(name, greeting):
    print(f"{greeting}, {name}!")

params = {"name": "Alice", "greeting": "Hello"}
greet(**params)          # Hello, Alice!
```

**对比 C#：** C# 的 `params` 只能收集位置参数（且必须同一类型）。Python 的 `*args` 收集为 tuple，`**kwargs` 收集为 dict，两种都能收。且 Python 支持**定义时收**和**调用时展开**双向操作。

### 工程要点：参数顺序

```python
def func(positional, *args, default="val", **kwargs):
    pass
# 顺序：位置参数 → *args → 仅关键字参数 → **kwargs
```

## 2. 作用域规则（LEGB）

### Python 的块级作用域问题

```python
if True:
    x = 42

print(x)  # 42 —— C# 里这样会编译错误！
```

**原因：Python 没有块级作用域。** 只有**函数级作用域**。

```csharp
// C#：块级作用域
if (true) {
    int x = 42;
}
Console.WriteLine(x);  // ❌ 编译错误：x 不在作用域中
```

### LEGB 查找规则

Python 在解析变量名时按四个层级查找：

```
L - Local（函数内部）
E - Enclosing（外层函数）
G - Global（模块级别）
B - Built-in（内置函数）
```

```python
x = "global"        # G：全局

def outer():
    x = "enclosing" # E：外层
    def inner():
        x = "local" # L：局部
        print(x)
    inner()

outer()             # 输出：local
```

### 为什么这样设计？

Python 的设计哲学是**简单**：
- 不需要用 `{ }` 创建作用域，用缩进就行
- 变量要么是函数局部，要么是全局/外层，没有"块局部"的概念

**代价：** 在 C# 里你在循环中创建临时变量不用担心泄漏到外面，但在 Python 中会：

```python
for i in range(10):
    temp = i * 2
    # 处理...

print(temp)  # 18 —— i 和 temp 都"泄漏"出循环了
```

这不是 bug，而是语言设计选择。你知道了这个规则就好。

## 3. `global` 和 `nonlocal`

### 为什么需要显式声明？

```python
count = 0

def increment():
    count += 1    # ❌ UnboundLocalError
    # Python 认为 count 是局部变量，但在赋值前被引用了
```

**Python 的规则：** 在函数内部给一个变量赋值，它就自动变成局部变量。

### 正确做法

```python
# global：声明我要修改全局变量
count = 0

def increment():
    global count
    count += 1

increment()
print(count)  # 1

# nonlocal：修改外层函数的变量
def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment
```

**设计权衡：** 为什么 Python 不自动判断？
- 如果你在函数里写 `count += 1`，Python 不知道你想修改全局的 `count` 还是创建一个同名的局部 `count`
- 显式声明避免歧义，也防止意外修改全局变量（这是设计选择——Python 倾向于显式而不是隐式）

**对比 C#：** C# 的 lambda 表达式直接捕获外部变量（闭包语义），不需要 `nonlocal` 声明。因为 C# 有块级作用域，变量在块内就是在块内，捕获就是捕获，没有歧义。Python 因为只有函数级作用域，引入 `nonlocal` 来区分"引用外层变量"和"创建新局部变量"。

## 4. `lambda` 表达式

### 基本用法

```python
# lambda：一句话的函数
add = lambda a, b: a + b    # 等价于 def add(a, b): return a + b
print(add(3, 5))             # 8

# 真正用途：作为参数传进去
pairs = [(1, "b"), (2, "a"), (3, "c")]
pairs.sort(key=lambda x: x[1])   # 按第二个元素（字符串）排序
print(pairs)                     # [(2, 'a'), (1, 'b'), (3, 'c')]
```

### 为什么 lambda 只能写一行？

**设计意图：** lambda 是用来定义**简单、临时、一次性**的函数的。

如果逻辑复杂到需要多行，就应该用 `def` 定义命名函数。

```python
# ✅ lambda：简单转换
names.sort(key=lambda s: s.lower())

# ❌ lambda：复杂的 lambda 不可读
names.sort(key=lambda s: s.split("_")[-1] if "_" in s else s)

# ✅ 改为 def
def extract_sort_key(s):
    if "_" in s:
        return s.split("_")[-1]
    return s

names.sort(key=extract_sort_key)
```

**对比 C# 的 lambda：** C# 的 lambda 可以多行（3.0 起），且支持 async await。Python 的 lambda 设计更克制——只做「一句话」的事。原因是 Python 用缩进定义代码块，多行 lambda 的语法会非常难看。

### lambda 的陷阱：闭包捕获

```python
funcs = []
for i in range(3):
    funcs.append(lambda: i)

for f in funcs:
    print(f())  # 2 2 2 —— 不是 0 1 2！
```

**原因：** lambda 捕获的是变量 i 的**引用**，不是创建时的值。循环结束后 i=2，所有 lambda 都看到 2。

**修复：**
```python
funcs = []
for i in range(3):
    funcs.append(lambda i=i: i)    # 用默认参数捕获当前值

for f in funcs:
    print(f())  # 0 1 2
```

## 5. 闭包（Closure）

### 函数为什么能"记住"外部变量？

```python
def make_multiplier(factor):
    def multiply(x):
        return x * factor
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))   # 10
print(triple(5))   # 15
```

`double` 和 `triple` 都是 `multiply` 函数，但它们的 `factor` 不同——各"记住"了创建时的值。

**本质：** 闭包 = 函数 + 捕获的外部环境。`double` 函数对象内部保存了一个对 `factor=2` 的引用。

**对比 C#：** C# 的 lambda 捕获也一样：

```csharp
Func<int, int> MakeMultiplier(int factor) => x => x * factor;
var double = MakeMultiplier(2);
Console.WriteLine(double(5));  // 10
```

**工程应用：**
- 工厂函数（如上）
- 装饰器（后续章节）
- 回调函数携带状态
