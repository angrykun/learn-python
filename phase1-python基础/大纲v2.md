### 1.4 面向对象（深度版）

**知识点：**
| # | 内容 | 深度方向 |
|---|------|---------|
| 1 | class & `__init__` | `__new__` vs `__init__` 的分工，创建对象的两个步骤 |
| 2 | `@property`（重点） | 为什么 Python 不需要 C# 的 get/set 方法？property 如何实现"看起来像属性" |
| 3 | `@classmethod` vs `@staticmethod` | 三种方法的区别和各自用途，C# 中的对应概念 |
| 4 | 特殊方法完整讲解 | `__str__`/`__repr__`/`__eq__`/`__hash__`/`__call__`/`__enter__`/`__exit__` |
| 5 | 继承与 MRO | C3线性化算法，`super()` 的调用链 |
| 6 | ABC 与抽象类 | 对比 C# 的 abstract class 和 interface |
| 7 | `@dataclass` | 自动生成了什么？和 C# 的 record 对比 |
| 8 | Mixin | 为什么 Python 需要这个模式？C# 里怎么模拟？ |
| 9 | `__slots__` | 为什么能省内存？限制了什么？ |
| 10 | Protocol（3.8+） | "结构子类型" vs "名义子类型"——C# 的接口是哪种？ |

**深度讲解方向：**
- `@property` 的本质：**Get 方法伪装成字段**，对比 C# 的自动属性
- `@classmethod` 和 `@staticmethod` 都是在创造**被类调用的函数**，不是实例调用的
- `__hash__` 和 `__eq__` 必须同时定义（和 C# 的 GetHashCode/Equals 完全一样）
- MRO 的真正用例：多重继承下的方法查找顺序
- ABC 和 Mixin 的边界：**ABC 强约束还是弱约束？**
